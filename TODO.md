## ✅ 완료된 작업 (2026-07-08) — 이메일 인증 도입

### 배경

- 구글 플레이 출시를 앞두고 보안 점검 중 이메일 인증이 꺼져 있어 아무 형식의 이메일로나 가입이 가능했던 점을 발견, 인증 활성화 결정
- 예전에도 시도했으나 Sender 주소가 `gmail.com`(Resend 미인증 도메인)이라 발송이 막혀서 방치했던 이력 확인 (Resend 로그에서 403 "Testing domain restriction" 원인 확인)

### 이메일 발송 인프라 구축

- Resend 가입, `firepath.me` 도메인 등록 및 GoDaddy 자동 연동으로 DNS(DKIM/SPF) 인증 완료
- Supabase Authentication > SMTP Settings에 Resend 연결 (Host: `smtp.resend.com`), Sender를 `noreply@firepath.me`로 수정
- 노출됐던 기존 API 키/미사용 "Onboarding" 키 정리
- "Confirm sign up" 이메일 템플릿에 로고(`logo-wide-nature.png`) 및 한글 안내 문구 추가
- URL Configuration의 Site URL을 `http://localhost:3000`(방치된 로컬 기본값) → `https://firepath.me`로 수정, Redirect URLs에 `https://firepath.me/**`, `http://localhost:3820/**` 추가 (인증 링크 클릭 시 "네트워크 서버에 연결할 수 없음" 오류 원인이었음)
- Authentication > Sign In/Providers에서 **Confirm email 활성화**

### 소스 수정

- 회원가입 성공 메시지를 "가입 확인 메일을 발송했습니다..."로 변경 (`LoginView.vue`)
- `email_not_confirmed` 에러 메시지를 "이메일 인증이 완료되지 않았습니다"로 명확화 (`errorMessage.ts`)
- 관리자 전용 RPC `admin_get_email_confirmations()` 신규 추가 — `auth.users.email_confirmed_at` 조회 (SECURITY DEFINER, 관리자 이메일만 호출 가능)
- 가입 이력 화면(`AdminSignupLogView.vue`)에 **활성 / 미인증 / 탈퇴** 3단계 상태 표시 추가
- 회원 현황 대시보드(`AdminView.vue`) 활성 회원 집계에서 미인증 계정 제외, 잔존율도 미인증 대기자를 분모에서 제외하도록 수정

## 💭 검토했으나 보류한 것 (나중에 다시 꺼내지 않도록 기록)

### 연도별 예상자산 추이 - 지난 연도 표시

- `AnalysisView.vue`의 "연도별 예상자산 추이" 카드는 `year = currentYear`부터만 루프를 도는 순수 예측(projection) 테이블. 템플릿/CSS에 `past` 상태(✓ 완료 뱃지 등) 스타일은 이미 있지만, 로직상 `year < currentYear`가 절대 참이 될 수 없어서 실제로는 사용되지 않는 죽은 코드임.
- "지난 연도도 보여주자"는 아이디어가 나왔었으나 **보류 결정**: 이 카드는 실제 과거 자산 기록을 저장/참조하지 않고 현재 자산 기준 미래 추세만 계산하기 때문에, 지난 연도를 단순히 "완료 100%"로 하드코딩하면 중간에 매도해서 자산이 줄어든 경우에도 항상 "완료"로 표시되어 **실제와 다른 잘못된 정보**를 보여주게 됨.
- 정확하게 하려면 `asset_history`(매일 자정 KST 실제 스냅샷) 데이터를 가져와 "그 시점 목표"와 비교해야 하는데, 목표/월 투자금/기대수익률을 나중에 바꾸면 "그때 계산했던 목표"가 무엇이었는지도 애매해져서 설계가 복잡해짐. 실익 대비 복잡도가 높다고 판단해 현재 방식(과거 연도는 표시하지 않고 그냥 리스트에서 빠짐) 유지하기로 함.

## ✅ 완료된 작업 (2026-07-04) — 글자 크기 조절 기능 + 모바일 버그 수정

### 글자 크기 조절 기능 신규 구현

- 더보기 > 화면 설정 메뉴 신설 — 슬라이더로 75%~150%(1% 단위) 조절, 100% 초기화 버튼
- `--font-scale` CSS 변수 + `useFontScale` 컴포저블 도입, 값은 로컬스토리지에 저장
- 프로젝트 전체 하드코딩된 `font-size: Npx`를 `rem`으로 전환 (30여개 파일) — 슬라이더 값 하나로 앱 전체 텍스트가 함께 커지고 작아짐
- 차트(SVG) 안 텍스트는 CSS가 아니라 태그 속성이라 이번 전환 대상에서 제외 (자산성장리포트/예측/FIRE시뮬레이터/ETF백테스트/FIRE히스토리 5개 화면 — 보류)
- 극단적으로 크게 설정했을 때 레이아웃이 깨지던 곳 보강: 자산·거래내역 헤더가 단어 중간에서 줄바꿈되던 문제, 요약 카드 값이 카드 밖으로 넘치던 문제, 종목명이 말줄임 없이 뚝 잘리던 문제
- 자산 요약 카드: 화면에 안 보이는 측정용 사본으로 실제 겹침 여부를 측정해서, 진짜 넘칠 것 같을 때만 매입금액/평가손익/평가금액/수익률 4칸이 동시에 라벨-값 스택 레이아웃으로 전환되도록 구현 (여유 있으면 항상 원래대로 한 줄 유지)

### 코드 정리

- 죽은 Vuetify 타이포그래피 클래스(`text-caption`/`text-body-1`/`text-body-2`/`text-h5`/`text-h6`) 제거 — 31개 파일, 303곳. 설치된 Vuetify 4(Material Design 3)에는 이 옛날 클래스명이 아예 존재하지 않아 원래도 스타일 효과가 없던 죽은 클래스였음 (제거해도 화면상 변화 없음)
- 미사용 디자인 토큰 `src/design/tokens/typography.ts`(fontSize/fontWeight) 삭제 — 정의만 되고 실제로 쓰는 곳이 없었음

### 모바일 pull-to-refresh 버그 수정

- 당겨서 새로고침 시 상단에 배경색 없는 검은 영역이 잠깐 보이던 문제 수정 (네이티브 오버스크롤 바운스가 커스텀 인디케이터와 같이 발생하던 것 — 당기는 순간에만 `preventDefault`로 차단)
- 위 수정 과정에서 생겼던 "화면 스크롤이 전혀 안 되던" 회귀 버그 재수정 (CSS `overscroll-behavior` 전역 차단 대신 제스처 발생 시점에만 조건부로 막는 방식으로 교체)
- 새로고침이 트리거되는 시점에 짧은 진동 피드백 추가 (Android Chrome만 동작 — iOS Safari는 Vibration API 자체를 지원하지 않아 홈 화면 추가해도 동작 안 함)
- 크롬 콘솔에 뜨던 "Ignored attempt to cancel a touchmove event" Intervention 경고 제거

### 스와이프 카드 모서리 비침 버그 재발 수정

- 이전 세션에서 고쳤던 자산·거래내역 스와이프 카드 모서리 비침이 재발 — 분수 배율(fractional device pixel ratio, 예: 크롬 반응형 모드)에서 래퍼와 카드가 각각 독립적으로 계산하는 클리핑 경계가 서브픽셀 단위로 어긋나는 게 근본 원인이었음
- 경계를 정밀하게 맞추려는 시도(반경 여유값, clip-path) 대신, 수정/삭제 버튼 영역 자체를 카드 경계보다 사방으로 6px 작게 배치하는 방식으로 견고하게 재수정 — 서브픽셀 오차가 나도 배경만 보이고 버튼 색은 안 비침

### 예측/자산성장리포트 화면 개선

- "연도별 예상자산 추이"에서 올해 목표 표시를 퍼센트 대신 다른 연도와 동일하게 금액으로 변경, 서브라벨 가독성 개선
- 자산성장리포트 차트의 "올해 목표"/"전체 자산" 라벨에 금액을 함께 표시, "전체 자산"→"현재 자산"으로 통일
- "막대 클릭 시 일별 보기" 안내 문구를 "👆 막대를 눌러 일별 증감 보기"로 개선
- 자산 요약 카드 "수익률(%)" 라벨을 "수익률"로 축약 (값 자체에 이미 %가 표시되어 중복)
