# Fire Path - 남은 작업 목록

## 🔴 해야 할 것

### 사용자 의견 보내기 시스템

**목표**: 게시판이 아닌 간단한 피드백 수집 창구. DB 저장 + 관리자 전용 관리.

#### 1. 더보기 메뉴
- "💬 의견 보내기" 항목 추가
- 설명 문구: "불편한 점이나 개선 아이디어를 보내주세요."

#### 2. 의견 보내기 화면
- 구분(필수): 버그 신고 / 기능 제안 / 기타 의견
- 제목(필수)
- 내용(필수)
- 이메일(자동 입력, 수정 불가)
- 하단 버튼: [의견 보내기]

#### 3. DB 테이블 (feedback)
| 컬럼 | 설명 |
|------|------|
| id | uuid PK |
| email | 작성자 이메일 |
| category | 버그신고 / 기능제안 / 기타의견 |
| title | 제목 |
| content | 내용 |
| created_at | 작성일 |
| status | NEW(기본값) / CHECKED / DONE |

#### 4. 관리자 페이지 (AdminFeedbackView)
- 관리 메뉴에 "💬 사용자 의견" 추가
- 목록: 최신순, 카테고리 / 제목 / 이메일 / 작성일 / 상태 표시
- 기능: 상태 변경, 삭제, 이메일/제목 검색

#### 5. 관리자 대시보드
- "미확인 의견 N건" 표시 (status = NEW 건수)

#### 6. 전송 완료 UX
- Toast 또는 Dialog로 감사 메시지 표시:
  "감사합니다. 보내주신 의견은 다음 업데이트에 적극 반영하겠습니다."

#### 제약사항
- 댓글, 좋아요, 공개 게시판 없음
- 기존 FIREPATH 디자인 스타일 유지
